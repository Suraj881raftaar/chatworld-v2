# API Contract Reference
## Chat World v2

This document defines the strict API contracts, request-response schemas, and typing models for all services.

---

### 1. Data Type Definitions (TypeScript vs Pydantic)

To maintain consistent typing across the stack, the following schemas are enforced:

| Field Name | Backend Type (Pydantic/Python) | Frontend Type (TypeScript) | Constraints |
| :--- | :--- | :--- | :--- |
| `id` | `uuid.UUID` | `string` (UUID v4 format) | Always generated by server |
| `email` | `pydantic.EmailStr` | `string` | Must be a valid email format |
| `username` | `str` | `string` | length: 3-50 characters |
| `role` | `enum.Enum` ("user", "admin") | `"user" | "admin"` | Default is "user" |
| `created_at` | `datetime.datetime` | `string` (ISO 8601 UTC) | e.g. "2026-07-19T13:20:00Z" |

### 2. HTTP Endpoint Contracts

#### 2.1 Endpoint: `POST /api/v1/auth/signup`
*   **Request Schema:**
    ```json
    {
      "email": "user@example.com",
      "username": "suraj",
      "password": "SecurePassword123!"
    }
    ```
*   **Response Schema (201 Created):**
    ```json
    {
      "id": "e30cb7be-3ab6-48c0-bc62-09439f07f4eb",
      "email": "user@example.com",
      "username": "suraj",
      "role": "user",
      "created_at": "2026-07-19T13:20:00Z",
      "updated_at": "2026-07-19T13:20:00Z"
    }
    ```

#### 2.2 Endpoint: `POST /api/v1/auth/login`
*   **Request Schema:**
    ```json
    {
      "email": "user@example.com",
      "password": "SecurePassword123!"
    }
    ```
*   **Response Schema (200 OK):**
    *   *Set-Cookie Header:* `refresh_token=<token_string>; HttpOnly; Secure; SameSite=Strict; Path=/api/v1/auth; Max-Age=604800`
    *   *JSON Body:*
        ```json
        {
          "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
          "token_type": "bearer",
          "user": {
            "id": "e30cb7be-3ab6-48c0-bc62-09439f07f4eb",
            "username": "suraj",
            "email": "user@example.com",
            "role": "user",
            "avatar_url": null,
            "status_message": null
          }
        }
        ```

#### 2.3 Endpoint: `GET /api/v1/rooms/{room_id}/messages`
*   **Query Parameters:**
    *   `limit`: integer, default `50`, maximum `100`
    *   `before`: string (UUID), optional
*   **Response Schema (200 OK):**
    ```json
    [
      {
        "id": "9a7bbd38-23ef-4b44-9988-cbfa389dcbb9",
        "room_id": "0dfbc398-aef4-4861-a083-d922bb37fb11",
        "sender_id": "e30cb7be-3ab6-48c0-bc62-09439f07f4eb",
        "message_type": "text",
        "content": "Hello from Chat World!",
        "file_url": null,
        "created_at": "2026-07-19T13:22:15Z",
        "updated_at": "2026-07-19T13:22:15Z"
      }
    ]
    ```

### 3. Error Responses Schema (Standardized RFC 7807 Exception Format)

All non-2xx HTTP responses must return the following JSON structure to allow simple client-side parsing:

```json
{
  "status_code": 403,
  "error_code": "INSUFFICIENT_PERMISSIONS",
  "detail": "You do not have the required permissions to delete this room.",
  "timestamp": "2026-07-19T13:24:00Z"
}
```

#### Common Error Codes:
*   `RESOURCE_NOT_FOUND` (404)
*   `INVALID_CREDENTIALS` (401)
*   `TOKEN_EXPIRED` (401)
*   `VALIDATION_ERROR` (400 - bad schema arguments)
*   `DUPLICATE_RESOURCE` (400 - username/email already exists)
